# Project Rules

## Language Guidelines

### UI Language
- **Polish only** - All user-facing text, labels, buttons, and messages must be in Polish
- No language switching functionality
- All UI strings should be clear and user-friendly for Polish speakers

### Code & Documentation Language
- **English only** - All code comments, function names, variable names, and documentation must be in English
- Follow standard English naming conventions for code
- Technical documentation should be written in English
- Commit messages should be in English

## Code Style

### General
- Use clear, descriptive variable and function names in English
- Add comments in English to explain complex logic
- Keep code modular and maintainable

### JavaScript
- Use modern ES6+ syntax
- Prefer `const` and `let` over `var`
- Use arrow functions where appropriate
- Add JSDoc comments for complex functions

### CSS
- Use CSS custom properties (variables) for theming
- Follow BEM-like naming conventions
- Keep selectors specific but not overly nested

### HTML
- Use semantic HTML5 elements
- Ensure accessibility with proper ARIA labels
- Keep markup clean and well-indented

## Project Structure
- Keep UI text in Polish
- Keep all technical implementation in English
- Maintain separation of concerns between presentation and logic
